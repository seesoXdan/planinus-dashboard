# 플래니어스 대시보드 · 설정 가이드

## 파일 구성 (다중 행사)

- **`index.html`** — 행사 허브. 모든 행사를 카드로 보여주고, 클릭하면 해당 행사 대시보드로 이동. **여기서 시작하세요.**
- **`0613event-dashboard.html`** — 행사 운영 대시보드(1개 파일). URL의 `?event=행사코드`로 행사를 구분합니다.

**핵심:** 행사마다 파일을 복사하지 마세요. 대시보드 파일은 하나만 두고, 허브에서 "＋ 새 행사 추가"로 행사를 늘립니다. 디자인을 고치면 모든 행사에 한 번에 반영됩니다. 각 행사의 데이터는 행사코드별로 따로 저장됩니다.

### 사용 흐름
1. `index.html`을 엽니다 → 비밀번호 입력(초기 관리자 `admin`).
2. 관리자 모드에서 **＋ 새 행사 추가** → 행사명·코드·일시·장소·색상 입력.
3. 카드를 클릭하면 그 행사 대시보드로 들어가고, 거기서 일정·자료 등을 채웁니다.
4. **한 번 로그인하면** 같은 창에서 행사들을 오갈 때 다시 로그인할 필요가 없습니다(세션 유지). 로그아웃하면 해제됩니다.

> 허브와 대시보드의 비밀번호는 **별도**입니다. 허브 비밀번호는 허브의 🔒 버튼에서, 각 행사 비밀번호는 그 행사 대시보드의 🔒 버튼에서 바꿉니다.
> Firebase로 실시간 공유하려면 **두 파일(index.html, 대시보드)에 똑같은 FIREBASE_CONFIG**를 넣어야 합니다.

## 0. 지금 바로 되는 것 (설정 없이)

파일을 더블클릭해 브라우저로 열면 **로그인 화면**이 먼저 뜹니다. 비밀번호로 모드가 갈립니다:

- **관리자**(초기 비번 `admin`): 모든 항목 입력·수정·삭제, 화면에 "관리자 모드" 표시
- **담당자**(초기 비번 `2025`): 보기 + 자료실 다운로드만 (모드 표시 없음)

편집한 내용은 **이 브라우저에만 저장**됩니다(우상단에 "로컬 저장"으로 표시). 담당자에게 진짜 실시간으로 공유하려면 **1번 Firebase 설정**이 필요합니다.

> 우상단 **로그아웃** 버튼으로 다시 로그인 화면으로 돌아갑니다.

### 비밀번호 변경
관리자로 로그인 → 우상단 **🔒 비밀번호 변경** 버튼 → 관리자/담당자 비밀번호를 새로 입력하고 저장하면 됩니다. (코드를 고칠 필요 없음) 변경된 비밀번호는 저장·동기화되며, 다음 로그인부터 새 비밀번호가 적용됩니다.

### ✨ AI 자동 채우기 (문서 분석)
관리자 모드에서 **✨ AI 자동 채우기** 버튼(또는 각 섹션의 ✨ 버튼) → 기획서·일정표 파일을 올리면 AI가 분석해 행사정보·추진일정·마일스톤·프로그램·연락처를 자동으로 채웁니다.

준비물 — **Anthropic(Claude) API 키** 1개:
1. https://console.anthropic.com 에서 가입 → API Keys에서 키 발급.
2. 대시보드에서 ✨ 버튼 → **🔑 AI 키 설정**에 붙여넣기. (이 키는 **관리자 브라우저에만 저장**되고 담당자·동기화에는 절대 포함되지 않습니다)
3. 파일 선택 → 범위(전체/항목별) 선택 → **분석 실행**.

지원 형식: PDF · 워드(docx) · 엑셀 · CSV · 텍스트. 한글(HWP)은 PDF로 저장해 올리세요. 스캔 이미지 PDF는 글자 추출이 안 됩니다.
주의: AI 결과는 문서마다 정확도가 다르니 **채운 뒤 반드시 확인·수정**하세요. 비용은 분석 1회에 수십 원 수준이며 사용량은 Anthropic 콘솔에서 확인됩니다.
> ⚠️ AI 기능은 `file://`로 직접 연 상태에서는 보안정책 때문에 호출이 막힐 수 있습니다. **웹에 올린(호스팅) 주소나 로컬 서버**에서 사용하세요.

### 자료실 파일 업로드 (최종본 교체)
관리자 모드에서 자료 카드의 ✎ → **첨부파일 업로드**로 파일을 올리면 그 항목의 다운로드가 해당 파일로 교체됩니다. 새로 올리면 최종본으로 갱신돼요.
- **로컬 저장 모드**: 약 4MB 이하 파일까지 업로드 가능
- **Firebase 모드**: 약 0.7MB 이하만 인라인 업로드(문서 용량 제한 때문). 그보다 큰 PDF·영상 등은 **공유 링크**(NAS/드라이브)를 쓰세요.
- 대용량 파일을 그대로 업로드해 쓰려면 **Firebase Storage** 연동이 필요합니다 — 원하시면 추가해 드릴게요.

## 1. Firebase 실시간 동기화 (관리자 수정 → 담당자 화면 자동 반영)

약 10분이면 됩니다. 무료 등급으로 충분합니다.

1. **프로젝트 생성** — https://console.firebase.google.com 접속 → "프로젝트 추가" → 이름 입력(예: planinus-dashboard) → 생성. (애널리틱스는 꺼도 됩니다)
2. **웹 앱 등록** — 프로젝트 첫 화면에서 `</>` (웹) 아이콘 클릭 → 앱 닉네임 입력 → "앱 등록". 그러면 아래 같은 **설정값**이 나옵니다:
   ```js
   const firebaseConfig = {
     apiKey: "AIza....",
     authDomain: "planinus-dashboard.firebaseapp.com",
     projectId: "planinus-dashboard",
     storageBucket: "planinus-dashboard.appspot.com",
     messagingSenderId: "123456789",
     appId: "1:123...:web:abc..."
   };
   ```
3. **Firestore 데이터베이스 만들기** — 왼쪽 메뉴 "빌드 → Firestore Database" → "데이터베이스 만들기" → **테스트 모드**로 시작(우선 동작 확인용).
4. **HTML 파일에 값 붙여넣기** — `0613event-dashboard.html`을 열어 위쪽 `FIREBASE_CONFIG` 부분을 2번에서 받은 값으로 교체합니다.
   ```js
   const FIREBASE_CONFIG = {
     apiKey: "여기에 붙여넣기",
     ...
   };
   ```
   저장 후 새로고침하면 우상단이 **"실시간 동기화 중"** (초록 점)으로 바뀝니다.
5. **파일 공유** — 이제 같은 HTML 파일을 담당자에게 주거나 웹에 올리면, 관리자가 수정하는 즉시 담당자 화면에 반영됩니다.

### ⚠️ 보안 (행사 끝나기 전 권장)
테스트 모드는 30일 후 잠기고, 누구나 쓸 수 있어 위험합니다. 행사 운영용이라면 **Firestore 규칙**을 "읽기는 모두 허용 / 쓰기는 금지"로 바꾸는 것을 권합니다. (담당자는 어차피 읽기만 필요) 규칙 적용 방법이 필요하면 말씀해 주세요 — 따로 안내해 드리겠습니다.

> 여러 행사를 각각 운영하려면 파일 안의 `DASHBOARD_ID` 값만 행사별로 다르게 바꾸세요(예: `"2026-conf"`).

## 2. 파일을 어디에 올려 공유하나 (호스팅)

담당자가 링크로 접속하게 하려면 HTML을 어딘가에 올려야 합니다. 쉬운 순서대로:

- **사무실 NAS 웹 호스팅** — 시놀로지/큐냅 모두 웹 호스팅 기능이 있어 `http://나스주소/dashboard.html` 형태로 띄울 수 있습니다.
- **무료 정적 호스팅** — Netlify, Vercel, GitHub Pages 등에 파일 하나만 올리면 됩니다.
- **그냥 파일 전달** — Firebase만 연결돼 있으면, 파일 자체를 카톡/메일로 보내도 각자 열어서 실시간 동기화됩니다.

## 3. 자료실 ↔ 사무실 NAS 연결 (6번 항목)

자료실의 각 버튼은 **"공유 링크"** 를 연결하는 방식입니다. 관리자 모드에서 자료 카드의 ✎ 버튼 → "공유 링크" 칸에 붙여넣으면 끝입니다.

### 시놀로지(Synology)
1. **File Station** 열기 → 공유할 파일/폴더 우클릭 → **"공유"**
2. 생성된 링크 복사 (예: `https://나스주소:5001/sharing/xxxxx`)
3. 외부(집·이동 중)에서도 열리게 하려면 **QuickConnect** 또는 DDNS가 켜져 있어야 합니다.
4. 그 링크를 자료실 ✎ → 공유 링크에 붙여넣기.

### 큐냅(QNAP)
1. **File Station** → 파일 우클릭 → **"공유 → 공유 링크 생성"**
2. 만들어진 링크 복사 → 자료실에 붙여넣기.
3. 외부 접속은 **myQNAPcloud** 설정이 필요합니다.

### 확인 필요 사항
- NAS가 **외부 인터넷 접속(QuickConnect/myQNAPcloud/DDNS)** 을 허용하고 있는지 — 안 돼 있으면 담당자가 사무실 밖에서 다운로드가 안 됩니다.
- 보안상 폴더 전체보다 **파일별 공유 링크** 또는 **만료기간이 있는 링크** 사용을 권장합니다.

> NAS 브랜드/접속 방식(외부 접속 여부)을 알려주시면, 그 환경에 맞춰 더 구체적으로 안내해 드리겠습니다.

## 4. 관리자 비밀번호 변경

평소에는 **관리자 모드 → 🔒 비밀번호 변경** 버튼으로 바꾸면 됩니다(권장).

비밀번호를 깜빡해서 **초기화**해야 한다면, `0613event-dashboard.html` 위쪽의 초기값을 고친 뒤 저장된 데이터를 비우면 됩니다:
```js
const INIT_ADMIN_PW = "admin";   // 최초 1회 기본값
const INIT_STAFF_PW = "2025";
```
> 이 비밀번호는 가벼운 보호입니다. 진짜 보안은 위 1번의 Firestore 규칙(쓰기 금지)으로 잠그는 것이 핵심입니다.
